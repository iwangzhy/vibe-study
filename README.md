# vibe-study

通过这个工程来实践 vibe-code.

为什么会选择社交媒体平台呢？ 因为是一个真实的高并发场景。

功能模块其实很简单，发帖，评论，点赞，feed 流，就这些。


通过开发一个社交媒体平台（类Twitter/微博）来巩固学习到微服务架构、高并发处理、分布式系统等核心技术

会使用到 AI 相关的工具，毕竟这是一个趋势。但是能保证的是，每行代码都会 review。

这里面所有的模块，都会“借鉴” X和微博。

- 用户模块
- 动态模块
- feed
- 评论
- 互动
- 搜索

这几个模块是重要，要先从这几个模块开始。

这些技术栈都能用上。

- 前后端分离（Vue 3 + TypeScript + Spring Boot）
- 微服务架构（Spring Cloud Alibaba）
- 缓存优化（Redis多种数据结构）
- 消息队列（Kafka异步处理）
- 全文搜索（Elasticsearch）
- 实时通信（WebSocket + Netty）
- 分布式事务（Seata）
- 任务调度（XXL-Job）
- 数据同步（Canal）
- CI/CD（Jenkins + K8s）
