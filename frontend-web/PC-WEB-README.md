# 前端项目 - PC Web 系统说明

## 项目定位

这是一个**PC端Web社交媒体系统**，而非移动端App。UI设计和交互方式针对桌面浏览器优化。

## 界面设计风格

### 布局特点

- **侧边栏导航**: 左侧固定导航栏，可折叠（类似Twitter/Discord PC版）
- **顶部搜索栏**: 包含搜索框、通知、消息和用户菜单
- **主内容区**: 中央内容区域，最大宽度1200px，居中显示
- **响应式设计**: 适配不同分辨率的桌面显示器（1920x1080, 1366x768等）

### 登录注册页面

- **左右分屏布局**: 
  - 左侧：品牌展示区（渐变背景、特性介绍）
  - 右侧：表单区（纯白背景、简洁表单）
- **适合大屏**: 利用桌面端的宽屏优势
- **小屏自适应**: <1024px时隐藏左侧，只显示表单

## 核心功能页面

### 已实现页面

1. **登录页** (`/login`)
   - 左右分屏布局
   - 账号密码登录
   - 记住我功能

2. **注册页** (`/register`)
   - 左右分屏布局
   - 手机验证码注册
   - 用户协议确认

3. **首页** (`/home`)
   - 动态信息流
   - 推荐内容

4. **探索页** (`/explore`)
   - 发现新内容
   - 热门话题

5. **通知页** (`/notifications`)
   - 系统通知
   - 互动提醒

6. **消息页** (`/messages`)
   - 私信对话
   - 聊天列表

7. **个人主页** (`/profile/:id`)
   - 用户信息展示
   - 动态列表

8. **设置页** (`/settings`)
   - 账号设置
   - 隐私设置

9. **搜索页** (`/search`)
   - 搜索结果展示
   - 筛选功能

## UI组件库

使用 **Element Plus** - 适合PC端的Vue3组件库

### 主要使用的组件

- `el-container/aside/header/main`: 布局容器
- `el-menu`: 侧边栏菜单
- `el-button`: 按钮
- `el-input`: 输入框
- `el-form`: 表单
- `el-dropdown`: 下拉菜单
- `el-dialog`: 对话框
- `el-card`: 卡片
- `el-avatar`: 头像
- `el-badge`: 徽标
- `el-empty`: 空状态

## 交互特性

### PC端特有交互

1. **鼠标悬停效果**: 按钮、链接、卡片都有hover状态
2. **键盘快捷键**: Enter提交表单
3. **侧边栏折叠**: 点击按钮折叠/展开
4. **弹窗对话框**: 发布动态、确认操作使用Dialog
5. **顶部固定导航**: 滚动时保持顶栏可见

### 导航方式

- **侧边栏菜单**: 主要页面导航
- **顶部搜索**: 快速跳转
- **用户下拉菜单**: 个人相关操作
- **面包屑**: 层级导航（待实现）

## 样式规范

### 色彩

- **主色**: #1890ff (蓝色)
- **渐变**: #667eea → #764ba2 (紫色渐变)
- **背景**: #f0f2f5 (浅灰)
- **文字**: #333 (深灰), #666 (中灰), #999 (浅灰)
- **边框**: #e8e8e8 (浅灰线)

### 尺寸

- **侧边栏**: 展开250px, 折叠64px
- **顶栏高度**: 60px
- **主内容最大宽度**: 1200px
- **按钮大小**: large (适合PC点击)
- **输入框大小**: large
- **圆角**: 8px (卡片), 4px (按钮)

### 字体

- **标题**: 24px-48px, bold
- **正文**: 14px-16px
- **小字**: 12px-13px
- **字体**: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto等系统字体

## 技术栈

- **Vue 3**: Composition API + `<script setup>`
- **TypeScript**: 类型安全
- **Vite**: 构建工具
- **Vue Router 4**: 路由管理
- **Pinia**: 状态管理
- **Element Plus**: UI组件库
- **SCSS**: CSS预处理器

## 与移动端的区别

| 特性 | PC Web | 移动 App |
|------|--------|----------|
| 布局 | 侧边栏 + 主内容 | 底部导航 + 主内容 |
| 导航 | 左侧菜单 | 底部Tab栏 |
| 交互 | 鼠标点击、悬停 | 触摸、滑动手势 |
| 尺寸 | 大按钮、宽间距 | 小按钮、紧凑布局 |
| 输入 | 键盘 + 鼠标 | 虚拟键盘 |
| 弹窗 | Dialog对话框 | ActionSheet/Popup |
| 尺寸单位 | px | rem/vw/vh |

## 开发运行

```bash
# 安装依赖
cd frontend-web
npm install

# 开发模式
npm run dev
# 访问 http://localhost:3000

# 生产构建
npm run build

# 预览生产构建
npm run preview
```

## 注意事项

1. 所有尺寸针对**桌面端**优化，不要使用过小的字体和按钮
2. 充分利用**宽屏空间**，采用多栏布局
3. 保持**导航清晰**，用户能快速定位功能
4. 注意**响应式**，但优先保证1366px以上的体验
5. 使用**现代浏览器**特性（Chrome, Firefox, Edge, Safari）
