# 3个月Java全栈开发学习计划

> 目标：在3个月内掌握高级Java全栈开发技能，通过社交媒体平台项目实战，成功应聘高级Java开发工程师岗位

---

## 📖 学习方法

### 核心理念
**项目驱动学习法**：通过开发一个完整的社交媒体平台（类Twitter/微博），在实战中掌握所有核心技术。

### 学习原则
1. **先实现功能，再优化性能**：不要追求完美，先把功能做出来
2. **动手为主，理论为辅**：代码量至少10000行
3. **每周总结复盘**：记录技术难点和解决方案
4. **循序渐进**：从单体应用 → 微服务架构 → 生产优化

### 时间分配（每天8-10小时）
- **理论学习**：30%（视频、文档）
- **动手实践**：60%（写代码）
- **复盘总结**：10%（笔记、博客）

### Week 7-8：Feed流 + 互动功能 + 消息队列

#### 学习目标
- 掌握Feed流推拉模式（核心难点）
- 实现点赞、评论功能
- 使用RocketMQ异步处理
- 解决高并发问题

#### 实战任务

**1. Feed流设计与实现（⭐核心难点）**

推拉结合模式：
- **推模式（Push）**：粉丝<1000，发布动态推送到粉丝收件箱（Redis List）
- **拉模式（Pull）**：粉丝>1000，用户访问时实时拉取关注人动态
- **推拉结合**：根据粉丝数动态选择策略

实现细节：
- 用户发布动态后，异步推送到活跃粉丝的Feed流
- 使用Redis List存储用户的Feed流（最多保留1000条）
- 实现下拉刷新、上拉加载更多
- 查看个人主页时间线

**2. 点赞功能**
   - 点赞/取消点赞
   - 防止重复点赞（Redis Set）
   - 点赞数实时更新
   - 点赞列表查询
   - Redis + Lua脚本保证原子性
   - 延迟双删策略（缓存一致性）

**3. 评论功能**
   - 一级评论（对动态评论）
   - 二级评论（对评论回复）
   - 评论列表（分页、排序）
   - 评论点赞
   - 评论数统计

**4. 转发、收藏功能**
   - 转发动态
   - 收藏动态

**5. 消息队列（RocketMQ）**
   - 发布动态 → 异步推送到粉丝Feed流
   - 点赞/评论 → 异步发送通知消息
   - 关注用户 → 异步发送通知
   - 消息幂等性（唯一消息ID + Redis去重）
   - 消息可靠性（ACK机制、重试）

#### 技术要点
- **Feed流架构**：推拉模式、推拉结合
- **Redis数据结构**：List（Feed流）、Set（点赞去重）、Sorted Set（热度排序）
- **RocketMQ**：普通消息、顺序消息、事务消息
- **缓存一致性**：延迟双删、Canal监听binlog
- **分库分表**：按用户ID哈希分表（动态表）
- **布隆过滤器**：防止缓存穿透
