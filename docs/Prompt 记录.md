# Prompt 记录

## 制定计划

根据技术架构文档，来制定一个为期三个月的学习计划。

在这期间只要求实现下面这几个模块，

- 用户模块
- 动态模块
- feed
- 评论
- 互动
- 搜索

所有的模块的实现，原型，都借鉴 X 和微博平台。

我的要求是这个学习尽可能的详细，并且跳过一些环境的部署，如 jdk按照，mysql，docker等部署。默认所有环境都已安装。

在这个执行计划的阶段不需要你提供具体的代码，你只用说明白任务和一些技术需要注意的地方即可。如实现点赞功能就需要说明在开发这个功能的时候需要注意的东西。

点赞功能
- 点赞/取消点赞
- 防止重复点赞（Redis Set）
- 点赞数实时更新
- 点赞列表查询
- Redis + Lua脚本保证原子性
- 延迟双删策略（缓存一致性）

计划的第一步是从初始化工程开始，包括前端后端。然后是用户模块，动态，feed，评论，互动，搜索。
